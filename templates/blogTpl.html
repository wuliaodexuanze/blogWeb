类的声明

类的声明有2种方式：

  • 传统方式使用构造函数来声明
  • es6中使用 class

传统方式

function Animal(){
    this.name = 'name';
}

ES6方式

class Animal {
    constructor(){
        this.name = 'name';
    }
}

生成实例

传统方式和ES6方式声明实例的方式是一样的。

new Animal();

技巧：
官方文档上说明，如果实例化对象的时候，构造函数没有参数，new后面构造函数的“括号”是可以不写的。

如何实现继承

实现继承的本质就是对象的 原型链。

继承有几种方式及优缺点

1、借助构造函数实现继承

function Parent(){
    this.name = 'parent';
}

function Child(){
    Parent.call(this);  // 主要原理在这句，这里是关键
    this.type = 'child';
}

原理：
修改父对象的 this 指向。

缺点：
这样做只是修改了 Parent 中 this 的指向，这样写，虽然能继承，但是 Parent 原型对象上的属性和方法 Child 是没有继承下来的。

代码中 say 方法是 Parent 原型对象上的，Child 是没能继承下来的。

2、借助原型链实现继承

这个方法弥补了第一种方法不能继承父对象原型对象上属性和方法的缺点。

function Parent(){
    this.name = 'parent';
}

function Child(){
    this.type = 'child';
}

Child.prototype = new Parent(); // 没有参数可以省略“()”，这句是关键，么有这句实现不了继承。

注意理清思路。

缺点：
因为 共用原型对象上的属性 ，所以如果一个对象修改了原型对象上属性，会影响使用这个属性的其他对象。

3、组合方式

这种方式就是将前面两种方式结合起来实现的。结合他们的优点，弥补不足。

function Parent(){
    this.name = 'parent';
    this.li = [1,2,3];
}

function Child(){
    Parent.call(this);
    this.type = 'Child';
}

Child.prototype = new Parent();

var s1 = new Child();
var s2 = new Child();
s1.li.push(4);
console.log(s1.li, s2.li);

结果：

[1,2,3,4]
[1,2,3]

缺点：
这样写也有缺点，在代码中使用了两次 Parent ，就相当于执行了两次。这样是完全没必要的。

优化代码1：

function Parent(){
    this.name = 'parent';
    this.li = [1,2,3];
}

function Child(){
    Parent.call(this);
    this.type = 'Child';
}

Child.prototype = Parent.prototype;  // 优化在这里

var s1 = new Child();
var s2 = new Child();
s1.li.push(4);
console.log(s1.li, s2.li);

这样写的思路是，Parent.call(this) 操作完之后继承了父对象构造体上的属性和方法，剩下的只需要将父对象的原型对象赋值给子对象的 prototype 即可。这样构造体的拿到了，原型对象上的拿到了，完成了继承。

优点：
避免了多次执行父级构造函数。

缺点：
此时如果要判断s1或者s2是由Child直接构造，是不行的，本身s1、s2都是由Child直接构造的，按照之前的方式使用 constructor 来进行判断，结果是Child或Parent都会返回true，原因 Child.prototype = Parent.prototype 这句，子对象和父对象的原型对象指向是一个。（s1的构造函数按照原型链应该是Child.prototype.contructor，即应该指向Child自身，但是结果却不是。）

综合下：就是构造函数的指向出了问题，所有的都指向一个父对象。

优化代码2：

function Parent(){
    this.name = 'parent';
    this.li = [1,2,3];
}

function Child(){
    Parent.call(this);
    this.type = 'Child';
}

Child.prototype = Object.create(Parent.prototype);  // 优化在这里
Child.prototype.constructor = Child;  // 必须加上这句，不然构造函数根据原型链还是会找到Parent上去。

var s1 = new Child();
var s2 = new Child();
s1.li.push(4);
console.log(s1.li, s2.li);
